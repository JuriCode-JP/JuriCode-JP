## なぜ税法を追加したか

JuriCode-JP の国内向けターゲットは当初「**警察 + 自治体 + 法律実務家**」の三本柱だったが、
2026-05-20 に **税理士 / 税務担当** を 4 本目の柱として明示的に追加した。

理由:

1. **税務アドバイザー (余合氏) との接続点**が既に存在し、ユーザー発見・テスター獲得の経路が短い
2. 税法は**条文数あたりの実務照会頻度**が他法分野より高く、参照基盤の価値が出やすい
3. 税法は**改正の追跡**が必須で、JuriCode-JP の version_date / amendments 設計が活きる
4. 「源内」(デジタル庁 Lawsy-Custom-BQ) のダウンストリームとしても、税務 AI は需要が高い

## Phase 1 (税法) のマイルストーン

- **2026-05**: 主要 4 法令 (国税通則法・法人税法・所得税法・消費税法) を fetch → parse → verify → validate

## 税法を扱う上での注意

- **通達** (国税庁長官通達等) は法令ではないが、税務実務では極めて重要。これらは別レイヤー (将来仕様) で扱う。
- 税法は**改正頻度が高い** (毎年度改正が通例)。`version_date` と `amendments` 履歴の保守が他法令以上に重要。
- **租税特別措置法** は対象外 (現時点). 1,000 条超かつ毎年改正で、保守工数がプロジェクトリソースを超える可能性が高い。需要が明確になってから検討する。
- 個別事案への税務アドバイスは扱わない (JuriCode-JP は法令データ基盤であり、税務相談ツールではない)。
