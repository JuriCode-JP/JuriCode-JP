# JuriCode-JP メタタグ標準語彙

**バージョン**: v0.1 (2026-05-18 初版)
**位置づけ**: 各条文ファイルの YAML frontmatter `tags:` フィールドで使う統制語彙
**関連**: [format-spec.md](./format-spec.md), [ir-spec.md](./ir-spec.md), [glossary.md](./glossary.md)

---

## 1. なぜ統制語彙が必要か

RAG の精度向上には **Chunk Augmentation**(チャンクへのメタタグ付与)が決定的に効くことが、業界の知見として整理されている。`tags:` を自由文字列のままにすると:

- 同義語の揺れ(「正当防衛」「緊急行為」「違法性阻却」)で検索性が落ちる
- 上位下位概念の整理ができない
- ベクトル検索とキーワード検索のハイブリッドが弱くなる

そこで JuriCode-JP は **5 種類のタグカテゴリ**を定義し、各条文に最低 1 個ずつ付与する規約を設ける。

---

## 2. タグ規約

### 2.1 命名規則

- **文字種**: 日本語(漢字・ひらがな・カタカナ)、または ASCII 小文字 + ハイフン
- **空白**: なし(`正当防衛` は OK、`正当 防衛` は NG)
- **接頭辞**: カテゴリを示すプレフィックスは付けない(`phase1-police` のような複合タグは例外)
- **複数語**: ハイフン区切り(`違法性阻却事由`、`手続的-基本権利`)

### 2.2 1 つの条文に必須カテゴリ

各条文の `tags:` には、以下 5 カテゴリから最低 1 個ずつタグを付与する。

| カテゴリ | 説明 | 必須 | 例 |
|---|---|---|---|
| **A. フェーズ** | JuriCode-JP のロードマップ上の位置づけ | ✅ | `phase1-police`, `phase2-civil` |
| **B. 法分類** | 法令の大分類(刑事/民事/商事/憲法/行政) | ✅ | `刑事法`, `民事法`, `憲法` |
| **C. 概念** | 条文が扱う中核概念 | ✅(1個以上) | `正当防衛`, `責任能力`, `共犯` |
| **D. 手続き** | 関連する手続き種別 | 任意 | `捜査`, `逮捕`, `公判` |
| **E. ステークホルダー** | 主体・対象 | 任意 | `警察`, `検察`, `市民`, `被疑者` |

## 5. 運用ルール

### 5.1 タグ追加のプロセス

1. 既存語彙で表現困難な概念に遭遇 → Issue で議論
2. 合意形成 → 本ファイル(`docs/tag-vocabulary.md`)に追記する PR
3. PR 通過後に対象条文の `tags:` に追加

### 5.2 検証ルール

`tools/validate/` で以下をチェック:
- ✅ 各条文に **カテゴリ A・B・C のタグが最低 1 個ずつ**ある
- ✅ すべてのタグが本ファイルの統制語彙集に存在する
- ⚠️ 警告: 1 条文のタグが 10 個超(過剰)、または 3 個未満(過少)

### 5.3 検索・RAG での活用

- **メタタグフィルタ検索**: ユーザが「正当防衛について教えて」と質問 → `正当防衛` タグを持つ条文だけに絞り込み → そこから cos 類似度で検索 → 精度向上
- **Top-K 戦略**: 同じタグを持つ条文を関連条文として複数同時取得
- **カテゴリ別ルーティング(Self-route)**: 質問内容から推定したカテゴリでフィルタした上で検索
